## Params
    
<ParamField path="key" type="string" required={true}>
  A unique string. Please see the [Keys and Resumability](/guides/resumability)
  doc for more info.
</ParamField><ParamField path="params" type="object" required={true}>
   Post Message Input
<Expandable title="properties"><ParamField path="channel" type="string" required={true}>
   Channel, private group, or IM channel to send message to. Can be an encoded ID, or a name. See [below](#channels) for more details.
</ParamField><ParamField path="as_user" type="string" required={false}>
   Pass true to post the message as the authed user, instead of as a bot. Defaults to false. See [authorship](#authorship) below.
</ParamField><ParamField path="attachments" type="string" required={false}>
   A JSON-based array of structured attachments, presented as a URL-encoded string.
</ParamField><ParamField path="blocks" type="string" required={false}>
   A JSON-based array of structured blocks, presented as a URL-encoded string.
</ParamField><ParamField path="icon_emoji" type="string" required={false}>
   Emoji to use as the icon for this message. Overrides `icon_url`. Must be used in conjunction with `as_user` set to `false`, otherwise ignored. See [authorship](#authorship) below.
</ParamField><ParamField path="icon_url" type="string" required={false}>
   URL to an image to use as the icon for this message. Must be used in conjunction with `as_user` set to false, otherwise ignored. See [authorship](#authorship) below.
</ParamField><ParamField path="link_names" type="boolean" required={false}>
   Find and link channel names and usernames.
</ParamField><ParamField path="mrkdwn" type="boolean" required={false}>
   Disable Slack markup parsing by setting to `false`. Enabled by default.
</ParamField><ParamField path="parse" type="string" required={false}>
   Change how messages are treated. Defaults to `none`. See [below](#formatting).
</ParamField><ParamField path="reply_broadcast" type="boolean" required={false}>
   Used in conjunction with `thread_ts` and indicates whether reply should be made visible to everyone in the channel or conversation. Defaults to `false`.
</ParamField><ParamField path="text" type="string" required={false}>
   How this field works and whether it is required depends on other fields you use in your API call. [See below](#text_usage) for more detail.
</ParamField><ParamField path="thread_ts" type="string" required={false}>
   Provide another message's `ts` value to make this message a reply. Avoid using a reply's `ts` value; use its parent instead.
</ParamField><ParamField path="unfurl_links" type="boolean" required={false}>
   Pass true to enable unfurling of primarily text-based content.
</ParamField><ParamField path="unfurl_media" type="boolean" required={false}>
   Pass false to disable unfurling of media content.
</ParamField><ParamField path="username" type="string" required={false}>
   Set your bot's user name. Must be used in conjunction with `as_user` set to false, otherwise ignored. See [authorship](#authorship) below.
</ParamField></Expandable></ParamField>
